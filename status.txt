Here's the full project summary:

Game: Zenrush
Top-down auto-attack shooter roguelite for Android (iOS secondary). Hard-capped 10-minute runs, mid-run upgrade selection every 60 seconds, persistent shard-based meta progression. Visual direction: minimalist, monochromatic dungeon themes inspired by Nodebuster. Primary references: Archero and Survivor.io. Core loop: fight escalating waves → collect shards → pick mid-run upgrades → bank shards → spend on permanent meta upgrades → repeat.

Lucas — Strengths & Weaknesses
Strengths: ~5 years backend experience, prior iOS and Flash game dev background. Strong architectural instincts, pragmatic, composition over inheritance, pushes back on over-engineering. Good at knowing when to park ideas.
Weaknesses: Excitement paralysis and pivot temptation when hitting uncomfortable phases. Tendency to generate creative escape routes when anxious. Managed best via structured planning and parking lot approach.

Architecture
Core Scenes

World — root scene, orchestrates via signals. Holds Player, SpawnManager, UpgradeManager, CanvasLayer, MetaProgressionManager
MainMenu — entry point, shows total shards
MetaUpgradeScreen — standalone scene with its own MetaProgressionManager

Player (CharacterBody2D)
Key properties: max_health, health_regen, move_speed, base_damage, attack_cooldown, attack_range_percent, vampirism, explosion_radius, explosion_damage_multiplier, shard_magnet, burst_count, extra_shots, projectile_upgrades, player_upgrades. Calls apply_meta_stats() at run start.
Enemies

BaseEnemy — base class with Hurtbox/Hitbox composition, health bar, shard drop, vampirism trigger, override_color for color inheritance
RusherEnemy — fast, direct chaser
TankEnemy — slow, high HP
RangedEnemy — keeps distance, shoots projectiles
BossEnemy — spawns in final minute, drops shard pile on death
ChargerEnemy — TRACKING → WINDUP → CHARGING → COOLDOWN state machine, scale tween as tell
SplitterEnemy — dies into N children, generic via split_count/split_into/scale_per_tier/drop_shards_on_split
BlasterEnemy — radial N-shot burst, one shot always aimed at player
SummonerEnemy — spawns capped minion count, color inherits from summoner
CasterEnemy — Line2D beam, locks on cast, black windup → orange fire
CircleCasterEnemy — circle spawned at locked player position, reparented to scene root during cast
HealerEnemy — heals nearby enemies, caches closest ally via died signal
OrbEnemy — N orbs orbiting around itself, damages player on contact

Projectile
BaseProjectile (Area2D) — shooter, speed, damage, pierce_count, explodes, explosion_radius, explosion_damage_multiplier. Spawns ProjectileExplosion on impact if explosive.
In-Run Upgrade Architecture
BaseUpgrade → BasePlayerUpgrade / BaseProjectileUpgrade. Concrete upgrades: AttackSpeed, Damage, MaxHealth, MoveSpeed, HealthRegen, Vampirism, AttackRange, Burst, MultiShot, OrbitalOrb, ShardMagnet, Pierce, Explosive, ProjectileSize. UpgradeManager owns pool, picks 2 random offers per trigger, depletes chosen upgrade for rest of run.
Meta Progression
MetaUpgrade (Resource) — id, name, description, category (SURVIVAL/ATTACK/FORTUNE), values_per_level, player_stat_key, requirements. Cost curve: base_cost(10) * growth(2.0)^(level-1). Nine upgrades across three trees with skill ladder requirements. MetaProgressionManager scans res://scenes/meta_upgrades/ at runtime.
Other Systems

SpawnManager — run timer, difficulty scalar, enemy spawning, boss spawn at final minute
SaveManager (autoload as Save) — ConfigFile persistence for total_shards and meta_levels
