Help me on my project.

Game: Zenrush
Description (subject to change and get small pivots): Top-down auto-attack shooter roguelite for iOS/Android. Hard-capped 10-minute runs, mid-run upgrade selection every 60 seconds, persistent shard-based meta progression. Visual direction: minimalist, monochromatic dungeon themes inspired by games like Nodebuster or Super Hexagon. Primary references: Archero, Survivor.io, Halls of Torment, Brotato. Core loop: fight escalating waves → collect shards → pick mid-run upgrades → bank shards → spend on permanent meta upgrades → repeat.

Lucas's Preferences
Game: Approachable difficulty, short sessions, addictive daily grind, Tibia-like endless progression
Code: Typed GDScript, composition over inheritance, no magic numbers, no unjustified singletons, clean separation of responsibilities, pushes back on over-engineering
Strengths: ~5 years backend experience, prior iOS and Flash game dev background. Strong architectural instincts, pragmatic, good at knowing when to park ideas
Weaknesses: Excitement paralysis and tangents — managed via structured planning and parking lot approach. Very critical of game design and overall gameplay feel.

Architecture
Core Scenes

World — root scene, orchestrates via signals. Holds Player, SpawnManager, UpgradeManager, CanvasLayer, MetaProgressionManager
MainMenu — entry point, shows total shards
MetaUpgradeScreen — standalone scene with its own MetaProgressionManager

Player (CharacterBody2D)
Key properties: max_health, health_regen, move_speed, base_damage, attack_cooldown, attack_range_percent, vampirism, explosion_radius, explosion_damage_multiplier, shard_magnet, burst_count, extra_shots, projectile_upgrades, player_upgrades. Calls apply_meta_stats() at run start.
Enemies
BaseEnemy — base class with Hurtbox/Hitbox composition, health bar, shard drop, vampirism trigger, override_color for color inheritance

Projectile
BaseProjectile (Area2D) — shooter, speed, damage, pierce_count, explodes, explosion_radius, explosion_damage_multiplier. Spawns ProjectileExplosion on impact if explosive.
In-Run Upgrade Architecture
BaseUpgrade → BasePlayerUpgrade / BaseProjectileUpgrade. Concrete upgrades: AttackSpeed, Damage, MaxHealth, MoveSpeed, HealthRegen, Vampirism, AttackRange, Burst, MultiShot, OrbitalOrb, ShardMagnet, Pierce, Explosive, ProjectileSize. UpgradeManager owns pool, picks 2 random offers per trigger, depletes chosen upgrade for rest of run.
Meta Progression
MetaUpgrade (Resource) — id, name, description, category (SURVIVAL/ATTACK/FORTUNE), values_per_level, player_stat_key, requirements. Cost curve: base_cost(10) * growth(2.0)^(level-1). Nine upgrades across three trees with skill ladder requirements. MetaProgressionManager scans res://scenes/meta_upgrades/ at runtime.

Other Systems
SpawnManager — run timer, difficulty scalar, enemy spawning, boss spawn at final minute
SaveManager (autoload as Save) — ConfigFile persistence for total_shards and meta_levels
